create an outstanding-tasks.md in same folder. To keep it lightweight and fast for the AI to read, the log should include exactly four core elements per entry.

What outstanding-tasks.md Should Include

Timestamp & Agent Name: Who did the work and when (e.g., ## [2026-07-02 09:45] - Agent: Claude-3.5).

The Intent (What changed): A one-sentence summary of the feature added or bug fixed.

The State (Files modified): A punchy bulleted list of the exact files touched.

The Handoff (Next step): Explicit instructions on what the next agent should do.


 outstanding-tasks.md Template

Copy and paste this exact template into your project root. Keep the most recent entries at the top of the file so the AI sees them first without scrolling through massive history. [1]

markdown

# Project Activity Log

## [YYYY-MM-DD HH:MM] - Agent: [Agent Name]

- **Task Completed:** [One sentence summarizing the goal achieved]

- **Files Modified:** 

  - `path/to/file1.ext` (Brief note on change)

  - `path/to/file2.ext`

- **Current State:** [Briefly note if the app is currently running, broken, or needs testing]

- **Next Step / Handoff:** [Direct instruction for the next agent]
